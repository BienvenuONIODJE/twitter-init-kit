## Task Execution Methodology

This twitter-init-kit project treats Twitter growth like engineering: **structured sprints, clear ownership, shipping criteria**.

### The PDCA Loop (2-Week Sprint Cycle)

- **Plan (Monday, Week 1)**: Define "Sprint Narrative" (e.g., "Focus on accuracy value prop"), draft core threads, identify "Hero Feature" to demo
- **Do (Week 1-2)**: Execute content calendar, run outreach, ship growth triggers
- **Check (Friday, Week 2)**: Review metrics. Did "Accuracy" narrative drive higher conversion than previous narrative?
- **Act (Weekend)**: Update spec.md with learnings. Kill underperforming formats. Double down on winners for next sprint.
